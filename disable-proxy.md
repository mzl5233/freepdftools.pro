# 禁用系统代理，解决本地代理问题

## 问题诊断

根据控制台截图和错误信息，您的网站请求正被重定向到本地代理服务器(127.0.0.1:7890)，这是导致网站显示空白页的根本原因。此代理服务器可能是：

1. 系统全局代理设置
2. 浏览器代理设置
3. 本地hosts文件映射
4. VPN或其他网络工具(如Clash, V2Ray等)的设置

## Windows系统代理设置检查与关闭

### 1. 检查系统代理设置

1. 按下`Win + I`打开Windows设置
2. 选择`网络和Internet` > `代理`
3. 检查`使用代理服务器`设置是否开启
4. 如果开启，请将其关闭，特别是如果显示`127.0.0.1`和端口`7890`

![Windows代理设置](proxy-settings.png)

### 2. 检查浏览器代理设置

**Chrome浏览器**:
1. 打开设置(右上角`⋮` > `设置`)
2. 搜索`代理`或导航到`系统` > `打开计算机的代理设置`
3. 确保未启用任何将流量定向到本地地址的代理

**Edge浏览器**:
1. 打开设置(右上角`...` > `设置`)
2. 导航到`系统和性能` > `系统` > `打开您的计算机代理设置`
3. 确保未启用任何将流量定向到本地地址的代理

### 3. 检查hosts文件

1. 以管理员身份打开记事本
2. 打开文件`C:\Windows\System32\drivers\etc\hosts`
3. 检查是否有包含`freepdftools.pro`的行
4. 如果有，将其删除或注释掉(在行首添加`#`)

### 4. 检查网络代理软件

如果您使用了以下软件，请检查其规则：

**Clash**:
1. 打开Clash控制面板
2. 检查`规则`或`代理规则`部分
3. 确保没有将`freepdftools.pro`定向到本地代理的规则
4. 尝试暂时关闭Clash，测试网站是否可以正常访问

**V2Ray/Shadowsocks等**:
1. 找到并打开相应的配置界面
2. 检查是否启用了系统代理或PAC规则
3. 暂时关闭代理软件，测试网站是否正常

## 临时解决方案

如果您无法立即修改上述设置，可以尝试以下方法：

1. 使用浏览器的隐私模式（不使用系统代理）访问网站
2. 使用不同的浏览器（可能未配置代理）访问网站
3. 使用手机数据网络（非WiFi）访问网站

## 验证方法

修改设置后，请检查控制台中的网络请求：

1. 打开浏览器开发者工具(F12)
2. 选择`网络`(Network)标签
3. 刷新页面
4. 查看资源请求的目标地址是否仍显示为`127.0.0.1:7890`

如果不再显示`127.0.0.1:7890`，说明系统代理已被正确禁用。
